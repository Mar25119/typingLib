# Выводы по работе

## 1. Что было изучено
Я познакомился с базовыми возможностями модуля `typing` в Python: научился аннотировать параметры функций, возвращаемые значения и переменные с помощью типов-подсказок. Изучил такие конструкции, как `List`, `Dict`, `Union` и современный синтаксис объединения типов (`int | str`). Также понял, как аннотации интегрируются с инструментами статического анализа (например, mypy) и средами разработки.

## 2. С какими трудностями столкнулся
Основная сложность — понимание различий между синтаксисом для Python <3.10 (`Union`) и ?3.10 (`|`). Также потребовалось уточнить, как правильно описывать словари с полями разных типов, чтобы сохранить читаемость и корректность. На начальном этапе путался в том, влияют ли аннотации на выполнение программы (оказывается — нет, только на анализ).

## 3. Что получилось удачно
Удалось создать чистый, самодокументируемый код с полными аннотациями, который проходит проверку mypy без ошибок. Пример использования наглядно демонстрирует работу функции, а структура типов сделана так, чтобы её легко было расширить (например, вынести тип `User` в отдельное определение).

## 4. Где инструмент уместен
Модуль `typing` особенно полезен в:
- Библиотеках и API, где важна ясность интерфейсов
- Командной разработке
- Проектах среднего и большого размера
- Образовательных целях — для формирования дисциплины типизированного мышления

## 5. Где инструмент неуместен
Использование `typing` может быть избыточным в:
- Очень простых скриптах (например, одноразовые парсеры)
- Прототипировании, где важна скорость написания, а не надёжность
- Проектах на старых версиях Python (<3.5), где поддержка ограничена

## 6. Итоговая оценка
Модуль `typing` — мощный инструмент повышения качества кода. Он не требует изменения логики программы, но значительно улучшает её восприятие и обслуживание. Буду использовать его во всех новых проектах.
